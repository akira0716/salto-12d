# 要件定義書

## 1. プロジェクト概要

- **プロジェクト名**: 社内備品貸出管理システム
- **作成日**: 2026年4月1日
- **背景・目的**:
  > 現状、備品の貸出に際してチャットによる個別調整と、手動での在庫確認が発生しており、双方に多大な工数がかかっている。システム化により、在庫の可視化と個体管理、申請・承認フローの簡略化を実現し、業務効率を向上させる。
- **概要説明**:
  > 社員がWeb上で備品の在庫を検索・申請し、管理者が承認・ステータス管理（個体ごとの状態管理）を行えるWebアプリケーション。

## 2. 利用者・ステークホルダー

- **利用者（ユーザー）**: 全社員（備品を借りたい人）
- **管理部門**: 総務・情報システム等の備品管理担当者

## 3. システム要件

### 3.1 機能要件

| No     | 機能区分 | 機能名                 | 内容                                                                                                                                                                         | 対象アクター |
| :----- | :------- | :--------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------- |
| **1**  | 認証     | ログイン機能           | 社員アカウント（ID/パスワード）での認証を行う。                                                                                                                              | 共通         |
| **2**  | 備品参照 | 備品一覧・検索         | カテゴリやキーワードで現在「利用可」な備品を探し、状況を確認する。                                                                                                           | 共通         |
| **3**  | 個別機能 | 備品の貸出申請（予約） | 借りたい備品（個体）と利用期間、利用目的を指定して申請を行う。                                                                                                               | 単社員       |
| **4**  | 個別機能 | 自身の貸出状況確認     | 自分が現在借りている備品（返却期限など）や過去の貸出履歴を確認する。                                                                                                         | 単社員       |
| **5**  | 管理機能 | 申請の承認・却下       | 届いた貸出申請を確認し、承認・却下を判断する。<br/>※却下時は「却下理由（コメント）」を入力可能。<br/>※実物異常発覚による却下と同時に、備品を「修理中」へステータス変更可能。 | 管理者       |
| **6**  | 管理機能 | 返却管理（受領・更新） | 返却された備品の実物を確認し、該当の貸出情報を「返却済」に、備品個体を「利用可」に戻す。                                                                                     | 管理者       |
| **7**  | 管理機能 | 備品マスタ追加・編集   | 備品１つ１つを「個体」として登録・管理し、各ステータス（利用可、貸出中、修理中、廃棄済等）を更新する。                                                                       | 管理者       |
| **8**  | 管理機能 | 備品分類マスタ管理     | 備品を分類するためのカテゴリ（PC本体、モニター等）を登録・編集する。                                                                                                         | 管理者       |
| **9**  | 管理機能 | 全体貸出状況・遅延確認 | 現在貸出中のすべての備品と利用者を一覧で確認し、期限超過がないか等を把握する。                                                                                               | 管理者       |
| **10** | 共通機能 | 通知機能               | 申請時、承認/却下時（却下理由含む）、返却期限前などにシステムやメール（将来拡張）で通知する。                                                                                | 共通         |

### 3.2 非機能要件

- **パフォーマンス**: 検索・更新処理は1秒以内で完了すること。
- **可用性**: 24時間365日利用可能（メンテナンス時を除く）。
- **セキュリティ**: 社員以外はアクセス不可。一般社員と管理者の権限による閲覧制限の徹底。
- **運用・保守**: データの定期バックアップ。操作ログ等の保持。

## 4. データ要件（ER図に基づく）

※詳細は `doc/02_ER図.md` を参照のこと。

- **利用者（USERS）**: ユーザーID、氏名、メールアドレス、パスワード、権限
- **備品分類（EQUIPMENT_CATEGORIES）**: 備品分類ID、カテゴリ名、説明
- **備品（EQUIPMENTS）**: 備品ID（個体識別用）、分類ID、個体ステータス、説明
- **貸出情報（LOAN_REQUESTS）**: 貸出ID、ユーザーID、備品ID、各種日付（申請/開始/終了/実返却）、ステータス、利用目的、却下理由（管理者コメント）
