# テーブル定義書

ER図（`02_ER図.md`）と整合性を合わせた最新のテーブル定義書です。

## 1. テーブル一覧

| No | テーブル名 | 論理名 | 説明 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | `USERS` | 利用者 | システムを利用する社員・管理者の情報を管理する | |
| 2 | `EQUIPMENT_CATEGORIES` | 備品分類 | 備品のカテゴリ（PC, モニター等）を管理する | 新規追加 |
| 3 | `EQUIPMENTS` | 備品 | 貸し出し対象となる備品の個体マスター情報を管理する | 個体管理用 |
| 4 | `LOAN_REQUESTS` | 貸出状態（申請情報） | 誰が何を借りるか（申請）、及び貸出・返却ステータスを管理する | |

---

## 2. テーブル詳細

### 2.1 テーブル名：`USERS`（利用者）

*   **説明**: 利用者の基本情報を管理するテーブル

#### カラム定義

| カラム名 | 型 | NULL | キー | 説明 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | int | No | PK | ユーザーID |
| `name` | varchar(100) | No | | 氏名 |
| `email` | varchar(200) | No | UQ | メールアドレス |
| `password` | varchar(255) | No | | ハッシュ化されたパスワード |
| `role` | varchar(20) | No | | 権限（一般社員 / 管理者） |

---

### 2.2 テーブル名：`EQUIPMENT_CATEGORIES`（備品分類）

*   **説明**: 備品のカテゴリを管理するテーブル（外部キー参照用）

#### カラム定義

| カラム名 | 型 | NULL | キー | 説明 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | int | No | PK | 備品分類ID |
| `name` | varchar(100) | No | | カテゴリ名（例: ノートPC、ケーブル） |
| `description` | text | Yes | | カテゴリに関する補足説明 |

---

### 2.3 テーブル名：`EQUIPMENTS`（備品）

*   **説明**: 備品の「個体」情報を管理するテーブル。同じ製品であっても個別にレコードを持つ。

#### カラム定義

| カラム名 | 型 | NULL | キー | 説明 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | int | No | PK | 備品ID（管理番号等） |
| `name` | varchar(100) | No | | 備品名 |
| `category_id` | int | No | FK | 備品分類（`EQUIPMENT_CATEGORIES.id` を参照） |
| `status` | varchar(50) | No | | 個体ステータス（利用可、貸出中、修理中、廃棄済） |
| `description` | text | Yes | | 備品ごとの固有説明（スペックやキズ等） |

#### 外部キー

| カラム名 | 参照先テーブル | 参照先カラム | 備考 |
| :--- | :--- | :--- | :--- |
| `category_id` | `EQUIPMENT_CATEGORIES` | `id` | 削除時は紐づく備品がある場合エラーとする（RESTRICT） |

---

### 2.4 テーブル名：`LOAN_REQUESTS`（貸出状態）

*   **説明**: 備品の貸出申請から返却に至るまでの状態と履歴を管理するテーブル。

#### カラム定義

| カラム名 | 型 | NULL | キー | 説明 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | int | No | PK | 貸出ID |
| `user_id` | int | No | FK | 利用し申請を行ったユーザーのID |
| `equipment_id` | int | No | FK | 対象となる備品個体のID |
| `request_date` | date | No | | 申請日 |
| `start_date` | date | No | | 利用開始日 |
| `end_date` | date | No | | 利用終了（返却）予定日 |
| `status` | varchar(50) | No | | ステータス（申請中、承認済、却下、返却済） |
| `purpose` | text | No | | 申請者が入力した利用目的 |
| `rejection_reason`| text | Yes | | 管理者が入力する「却下理由」（却下時のみ使用） |
| `return_date` | date | Yes | | 実際の返却処理を行った日 |

#### 外部キー

| カラム名 | 参照先テーブル | 参照先カラム | 備考 |
| :--- | :--- | :--- | :--- |
| `user_id` | `USERS` | `id` | |
| `equipment_id` | `EQUIPMENTS`| `id` | |
