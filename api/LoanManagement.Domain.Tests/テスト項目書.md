# LoanManagement.Domain テスト項目書

`LoanManagement.Domain` プロジェクトにおける単体テストの項目一覧です。各テストはドメインモデルのビジネスルールが正しく適用されているかを検証します。

## 1. 備品エンティティ (EquipmentTests)

| No. | テストメソッド名 | テスト内容 | 期待される結果 | 備考 (対象ルール) |
| :-- | :--- | :--- | :--- | :--- |
| 1.1 | `ChangeStatus_FromDisposed_ThrowsException` | 状態が「廃棄済」の備品に対し、別の状態（利用可 等）へ変更しようとする | `DomainException` が発生すること | 廃棄済からは状態変更不可 |
| 1.2 | `ChangeStatus_FromUnderRepairToLoaned_ThrowsException` | 状態が「修理中」の備品に対し、直接「貸出中」へ変更しようとする | `DomainException` が発生すること | 修理中から貸出中へ遷移不可 |
| 1.3 | `ChangeStatus_ValidTransition_UpdatesStatus` | 状態が「利用可」の備品を「貸出中」など、許可された正常な状態へ変更する | ステータスが正しく更新されること | 正常系の状態遷移 |

## 2. 貸出申請エンティティ (LoanRequestTests)

| No. | テストメソッド名 | テスト内容 | 期待される結果 | 備考 (対象ルール) |
| :-- | :--- | :--- | :--- | :--- |
| 2.1 | `Constructor_EndDateBeforeStartDate_ThrowsException` | 貸出申請作成時、利用終了希望日が開始日よりも前の日付でインスタンスを生成しようとする | `DomainException` が発生すること | 日付の前後関係 |
| 2.2 | `Approve_WhenNotPending_ThrowsException` | ステータスが「申請中」以外（承認済など）の貸出申請に対して、承認(Approve)処理を実行する | `DomainException` が発生すること | 承認は申請中のみ |
| 2.3 | `Reject_WhenNotPending_ThrowsException` | ステータスが「申請中」以外の貸出申請に対して、却下(Reject)処理を実行する | `DomainException` が発生すること | 却下は申請中のみ |
| 2.4 | `Reject_WithEmptyReason_ThrowsException` | 却下理由に空文字を渡して、却下(Reject)処理を実行する | `DomainException` が発生すること | 却下時は理由必須 |
| 2.5 | `Approve_Valid_UpdatesStatus` | ステータスが「申請中」の貸出申請に対して承認処理を実行する | ステータスが「承認済」になること | 正常系の承認処理 |
| 2.6 | `Reject_Valid_UpdatesStatusAndReason` | 理由を伴って、ステータスが「申請中」の貸出申請に対して却下処理を実行する | ステータスが「却下」となり、理由が保存されること | 正常系の却下処理 |

## 3. 貸出エンティティ (LoanTests)

| No. | テストメソッド名 | テスト内容 | 期待される結果 | 備考 (対象ルール) |
| :-- | :--- | :--- | :--- | :--- |
| 3.1 | `Constructor_DueDateBeforeLoanDate_ThrowsException` | 貸出作成時、返却予定日が貸出日よりも前の日付でインスタンスを生成しようとする | `DomainException` が発生すること | 貸出日と予定日の逆転 |
| 3.2 | `Return_WhenNotActive_ThrowsException` | ステータスが「貸出中(Active)」ではないデータに対して返却(Return)処理を実行する | `DomainException` が発生すること | 返却は貸出中のみ |
| 3.3 | `Return_Valid_UpdatesStatusAndReturnDate` | ステータスが「貸出中」のデータに対して、有効な日付で返却処理を実行する | ステータスが「返却済」になり返却日が記録されること | 正常系の返却処理 |
| 3.4 | `IsOverdue_WhenActiveAndPastDueDate_ReturnsTrue` | 貸出中で、現在の日付が返却予定日を過ぎている状況で遅延判定(IsOverdue)を実行する | `True` が返ること | 遅延判定（遅延している場合） |
| 3.5 | `IsOverdue_WhenActiveAndBeforeDueDate_ReturnsFalse` | 貸出中で、現在の日付が返却予定日以内の状況で遅延判定(IsOverdue)を実行する | `False` が返ること | 遅延判定（期限内の場合） |
| 3.6 | `IsOverdue_WhenReturned_ReturnsFalse` | 既に返却済のデータに対して、現在の日付が予定日を超えている状態で遅延判定を実行する | `False` が返ること | 今回の判定では返却済なら遅延と見なさない |

## 4. 貸出ポリシーサービス (LoanPolicyServiceTests)

| No. | テストメソッド名 | テスト内容 | 期待される結果 | 備考 (対象ルール) |
| :-- | :--- | :--- | :--- | :--- |
| 4.1 | `ValidateLoanRequest_DurationOver30Days_ThrowsException` | 貸出期間（開始日〜終了日）が最大日数（30日）を超える申請を検証する | `DomainException` が発生すること | 最大貸出期間チェック |
| 4.2 | `ValidateLoanRequest_DurationZeroOrNegative_ThrowsException` | 貸出期間が0日以下（開始日と終了日が同一、または逆転など）の申請を検証する | `DomainException` が発生すること | 最小貸出期間チェック |
| 4.3 | `ValidateLoanRequest_HasOverdueLoans_ThrowsException` | ユーザの現在の貸出一覧に、遅延している貸出が含まれる状態で新規申請を検証する | `DomainException` が発生すること | 遅延時の新規不可要件 |
| 4.4 | `ValidateLoanRequest_MoreThan3ActiveItems_ThrowsException` | ユーザが現在持つ「申請中」と「貸出中」の合計が上限（3件）に達している状態で新規申請を検証する | `DomainException` が発生すること | 同時貸出・申請上限の制限 |
| 4.5 | `ValidateLoanRequest_ValidRequest_DoesNotThrow` | 上記の制限をすべて満たさない正常な条件で貸出申請の検証を実行する | 例外が投げられず通過すること | ポリシーの正常系検証 |
